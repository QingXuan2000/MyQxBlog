## 简介

Vaultwarden 是一个轻量级的 Bitwarden 服务器兼容实现，适合部署在低配置设备上。通过自托管，您可以完全掌控密码数据，同时免费使用原本需要付费的高级功能（如 TOTP 两步验证、文件附件等）。

我个人认为，对于希望减少对第三方服务依赖的用户来说，自部署是一个值得尝试的方向。当然，这也意味着您需要自行承担数据备份和安全维护的责任，这一点请务必权衡。

本文提供一份极简部署方案，仅需 Docker 和一个域名（可选）即可运行。需要说明的是，不同操作系统和网络环境下可能会遇到一些意外问题，我的经验主要基于 Ubuntu 22.04，其他发行版可能需要微调，恕我无法保证完全通用。

## 准备工作

- 一台 Linux 服务器（最低配置：1核 CPU，256MB 内存，1GB 可用磁盘空间。但说实话，256MB 跑起来可能稍紧，我建议至少 512MB）
- 已安装 Docker（建议使用 Compose V2，即 `docker compose` 命令）
- （可选）一个域名及 HTTPS 证书（推荐使用 Let's Encrypt，免费且自动化）

## 快速部署（使用 SQLite）

创建目录并进入：

```bash
mkdir ~/vaultwarden && cd ~/vaultwarden
```

创建 `compose.yml`（Docker Compose 现在推荐用这个文件名，`docker-compose.yml` 也兼容）：

```yaml
services:
  vaultwarden:
    image: vaultwarden/server:latest
    container_name: vaultwarden
    restart: always
    environment:
      DOMAIN: "https://vault.example.com"          # 换成你的实际域名，末尾不要加斜杠（这一点很多人会忽略，我也是踩过坑才知道）
      SIGNUPS_ALLOWED: "true"                      # 首次部署时启用，注册完第一个账号后改为 false
      ADMIN_TOKEN: "your_very_strong_token"        # 用于访问 /admin 管理面板，建议用 openssl rand -base64 48 生成，不要用弱密码
    volumes:
      - ./data:/data
    ports:
      - "127.0.0.1:8080:80"                       # 仅监听本地，由反向代理对外暴露（我觉得这样更安全）
```

启动服务：

```bash
docker compose up -d
```

**关于 WebSocket 的说明**：以前老教程里都会让你额外映射 3012 端口，并配一堆复杂的 Nginx 规则。但其实从 Vaultwarden v1.29.0 开始，WebSocket 实时同步已经整合到主 HTTP 端口里了，3012 端口早已废弃，官方也计划在后续版本彻底移除它。所以新部署直接走 80 端口就行，反向代理配置反而比以前简单多了——这是我踩过坑后最想告诉你的一个变化。

## 首次使用

1. 通过 `http://你的服务器IP:8080` 访问网页端（初始未配置 HTTPS，仅用于注册。我其实不太推荐在公网直接用 HTTP 注册，但如果只是临时操作且内网环境，问题不大）。
2. 点击“创建账户”，输入姓名、邮箱地址、主密码（务必牢记且足够复杂。主密码一旦忘记，没有任何办法恢复，因为服务端根本没有存储）。
3. 注册成功后，立即关闭公开注册：编辑 `compose.yml`，将 `SIGNUPS_ALLOWED` 改为 `"false"`，然后重启服务：
   ```bash
   docker compose up -d
   ```
   （Docker Compose 会自动检测配置变更并重建容器，无需先执行 `down`。）

## 配置反向代理与 HTTPS（强烈推荐）

使用 Nginx + Let's Encrypt 提供加密访问。说实话，这一步是大多数问题的来源，需要仔细检查。

### 安装 Nginx 和 Certbot

```bash
sudo apt update && sudo apt install nginx certbot python3-certbot-nginx
```

### 创建 Nginx 配置

编辑 `/etc/nginx/sites-available/vault`：

```nginx
# 定义上游服务器，启用 keepalive 提升性能
upstream vaultwarden-default {
    zone vaultwarden-default 64k;
    server 127.0.0.1:8080;
    keepalive 2;
}

# WebSocket 支持：官方推荐用 map 来处理 Connection 头，否则 keepalive 会失效
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      "";
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    listen [::]:80;
    server_name vault.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    listen [::]:443 ssl;
    http2 on;
    server_name vault.example.com;

    # 证书路径由 Certbot 自动管理，不要手动指定其他路径。但如果你用其他工具生成证书，记得改路径。
    ssl_certificate /etc/letsencrypt/live/vault.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/vault.example.com/privkey.pem;

    # 附件上传默认最大 525MB，如果限制太小会上传失败，建议显式设置
    client_max_body_size 525M;

    # 全局代理头设置
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;

    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    location / {
        proxy_pass http://vaultwarden-default;
    }

    # 可选：为管理面板增加额外基础认证，除了 ADMIN_TOKEN 再多一层防护
    #location /admin {
    #    auth_basic "Private";
    #    auth_basic_user_file /path/to/htpasswd_file;
    #    proxy_pass http://vaultwarden-default;
    #}
}
```

启用站点并获取证书：

```bash
sudo ln -s /etc/nginx/sites-available/vault /etc/nginx/sites-enabled/
sudo certbot --nginx -d vault.example.com
```

Certbot 会自动修改 Nginx 配置并重载。如果出现错误，请检查域名 DNS 是否正确解析到服务器 IP。

**如果你之前照着旧教程配过 3012 端口和 `/notifications/hub` 规则，现在可以全部删掉了**。所有流量（包括 WebSocket 实时同步）都统一走 `location /` 转发到 8080 端口，Nginx 通过 `Upgrade` 头自动协商。我当初困惑了很久，以为 WebSocket 必须单独配端口，结果新版 Vaultwarden 已经帮我们把这件事简化了。

## 基本安全设置

- **禁用公开注册**：如上所述设置 `SIGNUPS_ALLOWED=false`
- **设置强 ADMIN_TOKEN**：使用 `openssl rand -base64 48` 生成，不要怕麻烦。新版本其实还支持 Argon2id 哈希，但随机字符串作为 Token 依然有效
- **开启管理面板的 IP 限制**（可选）：在 Nginx 配置的 `server` 块中添加：
  ```nginx
  location /admin {
      allow 你的可信IP/32;
      deny all;
      proxy_pass http://vaultwarden-default;
  }
  ```
  我个人觉得这一步对于个人部署来说不是必须的，因为 ADMIN_TOKEN 本身已经足够强，但多一层防护总是好的。

## 日常维护

### 更新 Vaultwarden

```bash
cd ~/vaultwarden
docker compose pull
docker compose up -d
```

有时候更新后可能会出现配置文件不兼容的情况，不过 Vaultwarden 团队做得很好，我还没遇到过严重问题。如果担心，可以先看 Release Notes。

### 定期备份（确保数据一致性）

备份前建议停止容器，避免直接打包正在写入的 SQLite 数据库：

```bash
cd ~/vaultwarden
docker compose stop vaultwarden
tar -czf vaultwarden_backup_$(date +%Y%m%d).tar.gz data
docker compose start vaultwarden
```

说实话，停止容器备份虽然最安全，但会造成几秒钟的服务中断。如果你的服务对可用性要求很高，可以考虑使用 sqlite3 的 `.backup` 命令在线备份。不过对于个人使用，停机几秒完全可以接受。

建议将备份文件加密后上传到云存储或另一台机器。这一点我承认自己经常偷懒，但理论上非常重要。

## 客户端连接

在 Bitwarden 官方客户端（浏览器扩展、手机 App）中，选择“自托管”并输入 `https://vault.example.com`，然后使用之前注册的邮箱和主密码登录。如果连接失败，优先检查防火墙和 Nginx 日志 —— 我遇到过很多次其实是 SELinux 或 ufw 没有放行端口。

## 结语

通过以上步骤，您已经拥有一个安全、私密且完全免费的密码管理服务。后续可以根据需要进一步配置 SMTP 邮件、文件大小限制等高级选项。自部署的核心是“数据自主”，请务必保管好您的服务器和备份。
