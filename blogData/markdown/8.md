> QxBlog v0.1.1 发布：引擎升级，阅读体验全面优化

QxBlog 迎来 **v0.1.1** 更新。

## 核心改动：Markdown 引擎从 featherdown 迁移至 unified

这是 v0.1.1 最关键的技术决策。

过去 QxBlog 使用 featherdown 处理 Markdown 渲染，虽然轻量，但在扩展性上逐渐遇到瓶颈。这次我们将其全面替换为基于 [unified](https://unifiedjs.com/) 的现代化文本处理流水线。

unified 的优势在于插件化架构：解析、转换、生成三个环节可以自由组合插件，像搭积木一样扩展语法支持。迁移完成后，不仅当下的渲染质量得到提升，也为后续引入更多 Markdown 扩展语法（比如自定义容器、脚注等）打下了坚实基础。

---

## v0.1.1 具体带来了哪些变化？

### 1. 代码块终于有语法高亮了

借助 unified 生态的语法高亮能力，文章内的代码块现在支持自动语言检测与着色。无论是 ` ```js ` 还是 ` ```python `，都能获得清晰易读的视觉呈现，技术类文章的阅读体验提升非常明显。

### 2. 搜索引擎改为直接查询本地 Markdown 文件

我们调整了搜索数据的来源。构建脚本现在会额外把每篇 Issue 的原始 Markdown 正文连同元数据一起，保存到 `blogData/markdown/{id}.md`。站内搜索不再走其他数据层，而是直接读取这些本地 `.md` 文件，查询链路更短，逻辑也更直接。

同时，这也意味着你的所有文章原文都以标准 Markdown 格式留在了仓库里。日后如果想备份、迁移或者批量处理，直接从 `blogData/markdown/` 目录取走即可，非常方便。

### 3. 排版与阅读体验优化

- **阅读宽度调整**：文章正文和列表页的最大宽度从 `800px` 改为 `72ch`。以字符数为基准的度量方式，对中文排版更友好，行长更适宜阅读，能有效减少视线跳跃带来的疲劳感。
- **目录样式重构**：DOM 生成的文章目录（TOC）重新设计了样式与交互，层级更清晰，锚点跳转更平滑。
- **UI 细节微调**：间距、字号、色彩对比度等做了小幅调整，整体视觉更干净统一。

### 4. 加载动画更连贯

加载动画改为硬编码且默认显示，等到 DOM 完全就绪后再自动隐藏。这消除了以往可能出现的白屏闪烁或布局跳动，页面从打开到可阅读的过程更加连贯自然。

## 现在的 QxBlog 是什么样的？

如果你还不熟悉这个项目，QxBlog 是一个基于 **GitHub Issues** 驱动的静态博客方案。你只需要在仓库里新建 Issue 写文章，GitHub Actions 就会自动构建并生成静态页面，配合 GitHub Pages 即可完成部署。

目前支持的核心能力包括：

- Issue 驱动发布文章
- 自动生成文章详情页与分类索引页
- 前端动态分页
- 全文搜索（基于本地 Markdown 文件）
- 明暗主题切换
- 响应式布局（桌面与移动端）

默认主题 **QxPaper** 的定位是"简洁阅读 + 轻量交互"，主打干净留白与卡片式信息组织，让文章内容本身成为页面的绝对主角。

## 如何开始？

如果你还没有部署，大致只需这几步：

1. Fork 或克隆仓库，确保包含 `.github/workflows/qxblog-build.yml`
2. 编辑 `config/siteConfig.json` 填写站点标题、描述、关于页等
3. 编辑 `config/buildConfig.json` 设置你的 GitHub 用户名（`author`）和分页参数
4. 在仓库 **Settings → Pages** 中启用 GitHub Pages，**Build and deployment** 的 Source 选择 **GitHub Actions**
5. 新建 Issue 写文章，提交后 Actions 会自动构建部署

更详细的配置说明和部署指南，请参考仓库内的 [README.md](https://github.com/qingxuan2000/QxBlog/blob/main/README.md)。
