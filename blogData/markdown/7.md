# QxBlog 部署指南

QxBlog 是一个面向技术写作的博客框架，默认主题为 **QxPaper**。

QxPaper 采用偏技术出版物的设计风格，强调排版、阅读节奏与长内容体验，而不是传统博客常见的卡片化 UI。

默认特性包括：

- 深灰冷白配色
- Typography-first 排版
- GitHub 风格代码高亮
- KaTeX 数学公式
- Unified Markdown 渲染管线
- 长文章阅读优化
- GitHub Pages 自动部署

---

# QxPaper 默认主题

QxPaper 是 QxBlog 内置的默认主题。

整体风格偏向：

- Research Terminal
- Technical Publishing
- Markdown-native
- Quiet Interface

适合：

- 技术博客
- 工程文档
- 数学内容
- AI / 系统设计
- 长篇文章
- 研究笔记

主题默认：

- 使用 72ch 阅读宽度
- 优化中英文混排
- 支持亮暗色切换
- 使用 GitHub 风格代码高亮
- 使用 KaTeX 渲染数学公式

# 配置 GitHub Pages

如果你的仓库名称不是：

```text
QxBlog
```

需要修改项目中的 `base` 配置。

例如：

```ts
base: "/your-repository-name/"
```

GitHub Pages 最终部署地址通常为：

```text
https://username.github.io/repository-name/
```

因此 `base` 必须与仓库名保持一致。

---

# 启用 GitHub Pages

进入仓库：

```text
Settings
→ Pages
```

在 Source 中选择：

```text
GitHub Actions
```

---

# 创建 GitHub Actions 工作流

创建文件：

```text
.github/workflows/deploy.yml
```

写入：

```yml
name: Deploy QxBlog

on:
  push:
    branches:
      - main

permissions:
  contents: write
  pages: write
  id-token: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9

      - name: Install dependencies
        run: pnpm install

      - name: Build
        run: pnpm build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```


# 常见问题

## 页面样式丢失

通常是 `base` 配置错误。

检查：

```ts
base: "/repository-name/"
```

是否与仓库名一致。

---

## Actions 没有运行

检查：

```text
Settings
→ Actions
```

是否允许 Workflow 运行。

---

## 数学公式不显示

确认依赖已正确安装：

```bash
pnpm install
```
