自部署 Vaultwarden 简明指南

2026-05-31 QingXuanJun
编程 学习 教程

简介

Vaultwarden 是一个轻量级的 Bitwarden 服务器兼容实现，适合部署在低配置设备上。通过自托管，您可以完全掌控密码数据，同时免费使用原本需要付费的高级功能（如 TOTP 两步验证、文件附件等）。

我个人认为，对于希望减少对第三方服务依赖的用户来说，自部署是一个值得尝试的方向。当然，这也意味着您需要自行承担数据备份和安全维护的责任，这一点请务必权衡。

本文提供一份极简部署方案，仅需 Docker 和一个域名（可选）即可运行。需要说明的是，不同操作系统和网络环境下可能会遇到一些意外问题，我的经验主要基于 Ubuntu 22.04，其他发行版可能需要微调，恕我无法保证完全通用。

准备工作

  • 一台 Linux 服务器（最低配置：1核 CPU，256MB 内存，1GB 可用磁盘空间。但说实话，256MB 跑起来可能稍紧，我建议至少 512MB）
  • 已安装 Docker（建议使用 Compose V2，即 docker compose 命令）
  • （可选）一个域名及 HTTPS 证书（推荐使用 Let's Encrypt，免费且自动化）

快速部署（使用 SQLite）

创建目录并进入：

mkdir ~/vaultwarden && cd ~/vaultwarden

创建 compose.yml：

services:
vaultwarden:
image: vaultwarden/server:latest
container_name: vaultwarden
restart: always
environment:
DOMAIN: "https://vault.example.com" # 换成你的实际域名，末尾不要加斜杠
WEBSOCKET_ENABLED: "true" # 显式启用 WebSocket
SIGNUPS_ALLOWED: "true" # 首次部署时启用，注册完第一个账号后改为 false
ADMIN_TOKEN: "your_very_strong_token" # 用 openssl rand -base64 48 生成
volumes:
- ./data:/data
ports:
- "127.0.0.1:8080:80" # 仅监听本地，由反向代理对外暴露

启动服务：

docker compose up -d

关于 WebSocket 的说明：从 Vaultwarden v1.29.0 开始，WebSocket 实时同步已整合到主 HTTP 端口，3012 端口已废弃。新部署直接走 80 端口即可。

Volume 配置推荐

虽然上述配置使用 ./data:/data 已经可以使用，但为了更好的性能、安全性和可维护性，建议考虑以下方案：

1. 使用 Docker 命名卷而非 bind mount

命名卷由 Docker 管理，权限问题更少，备份也方便：

volumes:
vaultwarden_data:

services:
vaultwarden:
...
volumes:
- vaultwarden_data:/data

volumes:
vaultwarden_data:

备份时可用 docker run --rm -v vaultwarden_data:/data -v $(pwd):/backup alpine tar czf /backup/vaultwarden_data_backup.tar.gz /data

2. 限制 SQLite 数据库的并发连接数（避免数据损坏）

在 environment 中添加：
DATA_DB_MAX_CONNS: "1" # SQLite 官方建议单写连接

3. 如果坚持使用 bind mount（./data:/data），建议设置主机目录权限

mkdir -p ./data
chown -R 65532:65532 ./data # Vaultwarden 容器内默认使用 uid 65532（非 root 用户）
chmod 750 ./data

这样可以防止容器以 root 运行（默认已是非 root，但确保挂载目录属主正确）。

4. 为数据目录启用 noexec、nodev 等挂载选项（仅限高级用户）

如果使用单独的数据分区，可以在 /etc/fstab 中添加挂载选项，但普通用户可忽略。

首次使用

  1. 通过 http://你的服务器IP:8080 访问网页端（仅用于注册，建议在内网操作）。
  2. 点击“创建账户”，输入姓名、邮箱地址、主密码（务必牢记且足够复杂。主密码一旦忘记，没有任何办法恢复）。
  3. 注册成功后，立即关闭公开注册：编辑 compose.yml，将 SIGNUPS_ALLOWED 改为 "false"，然后执行 docker compose up -d 重启服务。

配置反向代理与 HTTPS（强烈推荐）

安装 Nginx 和 Certbot

sudo apt update && sudo apt install nginx certbot python3-certbot-nginx

创建 Nginx 配置

编辑 /etc/nginx/sites-available/vault：

upstream vaultwarden-default {

server {

启用站点并获取证书：

sudo ln -s /etc/nginx/sites-available/vault /etc/nginx/sites-enabled/
sudo certbot --nginx -d vault.example.com

Certbot 会自动修改 Nginx 配置并重载。如果出现错误，请检查域名 DNS 是否正确解析到服务器 IP。

注意：如果你之前照着旧教程配过 3012 端口和 /notifications/hub 规则，现在可以全部删掉。所有流量统一走 location / 转发到 8080 端口。

基本安全设置

  • 禁用公开注册：如上所述设置 SIGNUPS_ALLOWED=false
  • 设置强 ADMIN_TOKEN：使用 openssl rand -base64 48 生成
  • （可选）管理面板 IP 限制：在 Nginx 配置的 server 块中添加：
    location /admin {
    allow 你的可信IP/32;
    deny all;
    proxy_pass http://vaultwarden-default;
    }
    注意：如果 IP 是动态的，可能把自己锁在外面，谨慎使用。

日常维护

更新 Vaultwarden

cd ~/vaultwarden
docker compose pull
docker compose up -d

定期备份（推荐安全方式）

cd ~/vaultwarden
docker compose stop vaultwarden
tar -czf vaultwarden_backup_$(date +%Y%m%d).tar.gz data
docker compose start vaultwarden

如果使用 Docker 命名卷，备份命令参考上文。

建议将备份文件加密后上传到云存储或另一台机器。

客户端连接

在 Bitwarden 官方客户端（浏览器扩展、手机 App）中，选择“自托管”并输入 https://vault.example.com，然后使用之前注册的邮箱和主密码登录。如果连接失败，优先检查防火墙和 Nginx 日志：

sudo tail -f /var/log/nginx/error.log

结语

通过以上步骤，您已经拥有一个安全、私密且完全免费的密码管理服务。后续可以根据需要进一步配置 SMTP 邮件、文件大小限制等高级选项。自部署的核心是“数据自主”，请务必保管好您的服务器和备份。
