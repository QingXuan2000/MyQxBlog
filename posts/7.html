QxBlog博客框架部署指南

QingXuanJun 2026-05-26 前端 编程 学习 教程

这篇文章记录如何把 QxBlog 从本地仓库部署到 GitHub Pages，并通过 Issue 驱动持续发布内容。

一、部署前准备

在开始前，确认以下条件已经满足：

  • 已有 GitHub 账号
  • 已创建并推送 QxBlog 仓库
  • 仓库中包含 .github/workflows/qxblog-build.yml
  • 已配置 config/buildConfig.json 与 config/siteConfig.json

其中 author 必须与实际发布文章的 GitHub 账号一致，否则工作流可能不会将 Issue 视为有效文章来源。

二、配置站点基础信息

主要修改两个配置文件：

  1. config/siteConfig.json：站点标题、描述、导航、关于页、页脚等展示内容
  2. config/buildConfig.json：构建行为、时区、分页、友链等运行配置

建议先完成这些配置，再进行首次部署，避免上线后出现占位文案。

三、开启 GitHub Pages

打开仓库设置页：Settings -> Pages

按下面设置：

  1. Build and deployment 选择 Action
  2. 点击保存

完成后，GitHub 会分配一个默认访问地址：

  • 项目页：https://<用户名>.github.io/<仓库名>/
  • 若仓库名是 <用户名>.github.io，则可直接使用根域名访问

四、通过 Issue 发布第一篇文章

QxBlog 的核心是 “Issue 即文章”。

发布流程：

  1. 打开仓库 Issues，新建 Issue
  2. 标题作为文章标题
  3. 正文使用 Markdown 编写文章内容
  4. 添加一个或多个标签（将作为分类）
  5. 提交 Issue

随后 GitHub Actions 会自动执行构建，更新：

  • blogData/articles.json
  • blogData/categories.json
  • articles/pages/{id}.html
  • categories/{label}/index.html

通常几分钟后即可在站点看到新文章。

五、常见问题排查

1) 发布了 Issue 但文章没出现

优先检查：

  • Actions 是否执行成功
  • Issue 作者是否等于 buildConfig.json 的 author
  • Issue 是否被误关、标签是否异常
  • 仓库默认分支是否与 Pages 配置分支一致

2) 首页能打开，文章详情 404

可能是构建未完成或 articles/pages/{id}.html 尚未生成。先检查最新一次工作流日志，再等待 Pages 刷新。

3) 样式错乱或资源加载失败

通常与路径有关，检查：

  • 仓库名是否变更
  • 是否把站点部署到了子路径但仍按根路径访问
  • 浏览器缓存是否未刷新（可强制刷新）

六、可选：绑定自定义域名

如果你希望使用自己的域名：

  1. 在 Settings -> Pages 填写 Custom domain
  2. 在 DNS 服务商处配置对应的 CNAME 或 A 记录
  3. 等待证书签发后启用 HTTPS

建议保留 GitHub 提供的默认域名作为回退入口，便于排障。

七、关于默认主题 QxPaper

QxBlog 默认主题是 QxPaper，强调简洁阅读体验：

  • 卡片化信息布局，突出文章内容
  • 支持目录、代码高亮、数学公式（KaTeX）
  • 支持全文搜索、分类浏览、前端分页
  • 内置明暗主题切换

常见定制入口：

  • css/default.css：视觉样式与主题变量
  • js/default.js：主题切换与基础交互
  • config/siteConfig.json：文案与导航内容

建议先配置内容（siteConfig.json），再调整样式（CSS），最后再改交互（JS）。

结语

如果你希望一个轻量、可维护、内容驱动的博客系统，QxBlog + GitHub Pages 是非常省心的组合。写作只需要专注 Issue 内容，构建与部署可以交给自动化流程。
